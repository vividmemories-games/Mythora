Philosophy adapted from Labyrinth Legends AB0/AB5 — **without** maze tilesets or citadel world packs.

## Principles

1. **Systems over bespoke art** — reusable frames, rarity borders, skill icons
2. **Readability first** — board tiles and HP/resources beat illustration detail
3. **Dusk identity** — follow [Theme](../02_Design_System/Theme.md)
4. **AI-assisted production OK** — human review required before ship
5. **Stable IDs & naming** — `hero_mage.png`, `enemy_goblin.png`, `tile_red.png`

## Planned asset domains (when needed)

| Domain | Examples |
|--------|----------|
| Tiles | Five resource colors (+ future obstacles/power-ups) |
| Heroes | Portrait / bust for roster and battle header |
| Enemies | Portrait + optional intent icon |
| UI | Panels, buttons, resource chips |
| VFX | Match clear, skill cast, hit (lightweight) |
| Marketing | Store screenshots (later) |

## Out of scope now

Full AB workshop chain, store marketing packs, per-world environment kits.

## When to expand

Start AB1-style production standards (sizes, export, naming) when the first real hero/enemy art pass begins — not before the battle loop is fun.
